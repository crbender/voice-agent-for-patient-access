# Constitution: Patient Access Voice Agent Demo

## Principles

1. **Executive clarity first**: every screen must help a CIO, CTO, or COO understand business value within 10 seconds.
2. **Healthcare-safe by default**: use synthetic data only, avoid PHI, avoid clinical determinations, and make escalation visible.
3. **Demo reliability over novelty**: the artifact must run locally in a browser without accounts, build tools, or network dependencies.
4. **Architecture honesty**: clearly label what is mocked and where Azure Voice Live API, grounded content, and function calling would connect.
5. **Recording-ready polish**: interactions must be deterministic, punchy, and readable in a 90-second screen recording.

## Non-negotiables

- No real patient data.
- No hidden external calls during demo playback.
- No custom server requirement for the base demo.
- Preserve Clawpilot theme variables and accessible contrast.
- Keep Azure integration points documented but disabled by default.
