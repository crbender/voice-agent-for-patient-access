# Contributing

This project is a sample application intended to demonstrate patient access voice-agent patterns in a safe, synthetic environment.

## Before You Contribute

- Use synthetic data only
- Do not introduce production credentials or secrets
- Do not add real patient data, PHI, or environment-specific endpoints
- Keep changes aligned with the sample's purpose: explainable, demo-ready, and safe by default

## Development Expectations

- Prefer small, focused pull requests
- Document user-visible changes in `README.md` when appropriate
- Keep the scripted demo path reliable even when live voice services are unavailable
- Preserve the server-side credential boundary for realtime flows
- Maintain accessible UI patterns and readable executive-demo presentation quality

## Pull Request Checklist

- Describe the problem being solved
- Summarize the user-visible change
- Note any Azure or model assumptions
- Confirm no secrets or sensitive data were added
- Include screenshots when UI behavior changes
- Update documentation when setup, architecture, or safety boundaries change

## Code of Conduct

Please review [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.
